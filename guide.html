Guide

After having familiarized yourself with Ema by following the earlier section, you are now ready to dive deep into learning how to achieve specific things.

  • Defining your model – Define your site model such that it supports hot reload
  • Working with routes – Unless you site has a single page (index.html), you will need to manage a set of routes
  • Defining Ema instance – Constrain your model and route to work with static sites
  • Rendering HTML – You could use plain strings to build HTML, or use templates, or use one of the delightful Haskell DSLs

Beyond the model and route types, Ema leaves it up to you as to how to develop your site. The following are not required when using Ema; nevertheless they are useful inasmuch as they capture common patterns in writing a good static site:

  • Tailwind + Blaze – We recommend–but not mandate–Tailwind for CSS and blaze-html as HTML DSL
  • Working with files – Ema provides a helper to support hot-reload on files
  • Converting Markdown – Pointers on how to work with Markdown files
